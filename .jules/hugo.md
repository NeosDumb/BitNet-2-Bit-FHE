## 2024-05-24 - [SIMD Fused Quantization: The Calculus of Memory]
**Learning:** In high-throughput quantization kernels (`quantize_i2_s`), the conservation of memory bandwidth is as critical as the conservation of energy. The original implementation violated this by using a two-pass approach with an intermediate `malloc`/`free` buffer (`q8`). This created unnecessary entropy (heap allocation overhead) and cache pressure (writing to memory only to read it back immediately). By fusing the max-scan, quantization, and bit-packing into a single pass using AVX2 SIMD registers, we eliminated the intermediate state entirely. This is analogous to a conservative vector field where the path taken doesn't dissipate energy—we transformed the data directly in registers. The result was a ~16x speedup, proving that the most efficient algorithm is often the one that touches memory the least.
**Action:** When optimizing data transformation kernels, always look for opportunities to fuse operations and keep data in registers. Avoid intermediate heap allocations in hot loops like the plague. Use SIMD not just for parallel arithmetic, but for parallel bit-manipulation and packing. Crucially, always guard SIMD-specific intrinsics (like AVX2) with appropriate preprocessor checks and provide fallback implementations to maintain compatibility with older architectures (conservation of compatibility).
