## 2026-03-22 - O(1) Memory Mapping via Sign Invariance in Quantization
**Learning:** In scale-based quantization routines (e.g., `quantize_i2_s`), if the scale mathematically cannot be negative (as is the case when scale is derived from a `fmax(..., fabs(...))` max-scan), the algebraic evaluation `val * scale > 0` mathematically simplifies to `val > 0`. This sign invariance eliminates the temporal dependency of the quantization logic on the calculation of the global scale, revealing a hidden parallelizability between finding the scale and assigning bits.
**Action:** Always check quantization operations for temporal dependencies on scaling factors. When scales are strictly non-negative, leverage sign invariance to fuse max-scans and bit-packing operations, effectively reducing O(n) heap allocations (`malloc`) to O(1) memory mappings.
