## 2024-05-18 - Sign Invariance for O(1) Memory Mapping
**Learning:** In scale-based quantization implementations (like `quantize_i2_s`), computing an initial scale based on maximum magnitude inherently produces a non-negative scale factor (`scale >= 0`). Therefore, the check `val * scale > 0` mathematically simplifies to `val > 0` regardless of the scale's magnitude. This sign invariance removes the temporal dependency on calculating the global maximum scale before quantization begins.
**Action:** By applying this insight, multiple O(N) heap allocations and multiple passes over the input array were eliminated. By evaluating `val > 0` directly, max-scan and quantization can be successfully fused, allowing the data structure to follow an O(1) memory mapping strategy across all core CPU architectures (AVX, NEON).
