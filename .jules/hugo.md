## 2024-05-18 - [O(1) memory mapping via sign invariance allowing temporal decoupling]
**Learning:** In scale-based quantization (like `quantize_i2_s`), calculating quantized values often depends on a global scale derived from the maximum absolute value in the tensor. Naive implementations loop twice: once to find the max scale, and again to quantize. However, when the quantization logic involves checking the sign scaled by the maximum value (e.g., `(double)src[i] * max > 0`), the non-negativity of `max` mathematically guarantees that this expression is strictly equivalent to `src[i] > 0`. This sign invariance temporally decouples the quantization operation from the global scale calculation.
**Action:** By applying this mathematical insight, I can fuse the max-finding loop, the quantization loop, and the bit-packing loop into a single pass. This entirely eliminates the need for temporary heap allocations (such as the `q8` buffer of size $N$), replacing an O(N) memory requirement with O(1) memory mapping, while reducing cache misses and lowering computational latency. I will look for sign invariance and temporal decoupling in other numerical algorithms to eliminate redundant loops and memory allocations.
