## 2024-05-24 - [Scale-Invariant Fused Quantization]
**Learning:** In the `quantize_i2_s` algorithm, the ternary mapping $T(x) \in \{0, 1, 2\}$ traditionally relied on the maximum absolute scale of the tensor $S = \max(|x_i|)$. However, mathematically, for any non-zero element, the condition $x_i \cdot S > 0$ is exactly equivalent to $x_i > 0$ because $S$ is strictly positive. This reveals a critical property: the ternary mapping is structurally scale-invariant. Because of this, the computation of the maximum scale and the mapping process do not need to be sequential. They can be mathematically commuted and fused into a single O(n) pass over the data, completely eliminating the need for intermediate O(n) heap memory allocation (`malloc`/`free`) to store states before packing.
**Action:** Always inspect the mathematical dependencies of a loop. **Learning:** Mathematical operations must be perfectly aligned with the numerical data type of their source and destination boundaries to minimize computational action. In `quantize_i2_s`, scaling factors and data were explicitly single-precision (`float`), but intermediate threshold checks `(double)fabs(val)` upcasted evaluation to `double`. This created completely unneeded and invisible performance friction via register stalls and pipeline upcasting/downcasting steps, which in hot tight SIMD loops drastically degrades performance for zero added numerical accuracy.
**Action:** Always maintain strict numerical precision homogeneity across performance-critical mathematical loops. Audit code for implicit type promotion across thresholds (`fmax` vs `fmaxf`, `fabs` vs `fabsf`) and bind arithmetic specifically to the native storage types (e.g. `1e-6f`).
