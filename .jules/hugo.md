## 2024-05-24 - [O(1) Memory Fused Quantization Loop]
**Learning:** [The mathematical transformation of max-finding and quantization passes. Instead of making three O(N) linear passes (max-search, thresholding/quantizing to an intermediate O(N) byte buffer, and bit-packing), these operations can be mathematically fused into a single O(N) loop with O(1) memory overhead. This behaves like a conservation law in physics, minimizing the computational 'action' and eliminating the cache thrashing caused by the large intermediate allocations (`malloc(n * sizeof(uint8_t))`).]
**Action:** [Apply fused-loop transformations whenever a reduction step (like max-finding) and an element-wise mapping (like threshold quantization) happen sequentially. Combine passes and eliminate intermediary heap allocations to optimize CPU caching behavior.]
