## 2024-03-23 - Fused Quantization Loop
**Learning:** In scale-based quantization (`quantize_i2_s`), the output value calculation (`val * scale > 0`) is mathematically equivalent to checking the sign of the original value (`val > 0`) because `scale` is guaranteed to be non-negative (it's the maximum absolute value). This sign invariance implies that the global scale calculation is not temporally dependent for the core quantization logic!
**Action:** Replace the 2-pass quantization approach (which calculates max in pass 1, then allocates memory and bitpacks in pass 2) with a fused 1-pass loop. This eliminates the intermediate memory allocation entirely (O(N) -> O(1) auxiliary memory), reducing memory pressure and memory bandwidth overhead. The scale can be computed alongside the bitpacking.
